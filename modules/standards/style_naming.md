# Style And Naming

Project preferences focused on conventions not enforced by linters.

- File naming: snake_case (`data_loader.py`).
- Whitespace: one blank line between logical blocks; two between top-level classes/functions.
- Comments: focus on "why," not "what." Omit redundant comments.
- Conciseness: prefer `user` over `user_object_from_api`.
- Docstrings: keep brief; include params and return types.
- General: choose simple, readable code and modern language syntax.
